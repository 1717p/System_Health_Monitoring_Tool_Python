# PySpark ETL and CI/CD Pipeline

## Project Overview

This project demonstrates a basic ETL (Extract, Transform, Load) pipeline built using PySpark for processing structured datasets. The pipeline performs data cleaning, validation, transformation, and aggregation operations on transactional data while simulating distributed data processing workflows.

The project also integrates a simple CI/CD workflow using GitHub Actions to automate pipeline execution and maintain version-controlled deployment practices.

---

## Objectives

* Process and analyze structured datasets using PySpark
* Perform data cleaning and validation
* Apply transformation and aggregation operations
* Simulate ETL and distributed data processing workflows
* Implement automated CI/CD workflow using GitHub Actions

---

## Technologies Used

* Python
* PySpark
* SQL Operations
* Git & GitHub
* GitHub Actions (CI/CD)


---

## Key Features

* Data ingestion and preprocessing using PySpark
* Data validation and cleansing operations
* Aggregation and analytical processing using DataFrames
* Modular pipeline structure for maintainability
* Automated workflow execution using GitHub Actions

---

## Workflow Performed

1. Load structured transaction dataset
2. Validate and clean missing records
3. Transform and aggregate transaction data
4. Generate analytical summaries
5. Execute automated workflow through CI/CD pipeline

---

## Business Relevance

This project demonstrates foundational data engineering concepts used in enterprise environments, including ETL processing, large-scale data handling, automation workflows, and data quality validation.

It also highlights how distributed data processing frameworks like PySpark can be used to support scalable analytics and reporting workflows.

---

## Learning Outcomes

* Hands-on exposure to PySpark DataFrames
* Understanding of ETL and distributed processing concepts
* Experience with data validation and transformation workflows
* Familiarity with CI/CD automation using GitHub Actions
* Improved understanding of structured data pipeline design
